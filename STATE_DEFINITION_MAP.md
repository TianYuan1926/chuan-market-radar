# STATE_DEFINITION_MAP

本文件定义 Market Radar 的统一状态语义。代码事实源为 `src/lib/signal-state-semantics.ts`。

## 总原则

- 前端只能展示后端状态，不能把状态升级。
- 候选、观察、等待都不是交易信号。
- 只有 `TRADE_PLAN_READY` 可以进入狙击榜。
- `TRADE_PLAN_READY` 仍只代表人工计划复核入口，不代表自动下单。

## 状态表

| 状态 | 中文显示 | 唯一含义 | 是否可展示 | 是否可交易 | 是否可进狙击榜 | 是否可附完整计划 | 升级规则 |
| --- | --- | --- | --- | --- | --- | --- | --- |
| `WAIT` | 等待确认 | 有观察价值，但缺突破、回踩、反抽、量能或高周期确认 | 是 | 否 | 否 | 否 | 必须由后端重新通过证据融合、RR 和风控门禁 |
| `WATCH` | 只观察 | 只用于背景、跟踪或复盘 | 是 | 否 | 否 | 否 | 必须重新进入扫描和分析链路 |
| `CANDIDATE` | 候选验证 | 轻扫、榜单或深扫排队发现的待验证标的 | 是 | 否 | 否 | 否 | 必须完成深扫、结构、证据、RR 和风控 |
| `SIGNAL` | 信号观察 | 用户可读的观察归类，不代表可交易 | 是 | 否 | 否 | 否 | 必须升级到 `TRADE_PLAN_READY` 才能附计划 |
| `EVIDENCE_SIGNAL` | 证据观察 | 已有结构或数据证据，但交易条件未全部通过 | 是 | 否 | 否 | 否 | 必须由后端确认 RR、风控、结构、触发、失效条件全部满足 |
| `TRADE_PLAN_READY` | 交易计划就绪 | 证据、结构、RR、风控、入场、止损、目标和失效条件已由后端生成 | 是 | 人工复核后才可考虑 | 是 | 是 | 终态由后端策略层生成，前端无权升级 |
| `BLOCKED` | 不可交易 | 被风控、RR、结构、多周期、数据质量、晚到或失效条件拦截 | 是 | 否 | 否 | 否 | 阻断原因解除并重新通过完整链路 |

## 成熟度到状态映射

| 后端成熟度 | 统一状态 | 展示区域 | 用户边界 |
| --- | --- | --- | --- |
| `LIGHT_SCAN_MARK` | `CANDIDATE` | 覆盖/发现/候选统计 | 只做调度输入，不进入主信号区 |
| `DEEP_SCAN_CANDIDATE` | `CANDIDATE` | 候选验证区 | 只说明值得验证，不能当作交易信号 |
| `EVIDENCE_SIGNAL` | `EVIDENCE_SIGNAL` | 主信号/重点观察 | 可观察，不能交易，不能进狙击榜 |
| `REVIEW_ONLY` | `WATCH` | 复盘观察区 | 只做复盘教材，不追涨追跌 |
| `TRADE_PLAN_READY` | `TRADE_PLAN_READY` | 狙击榜 | 唯一允许展示完整计划 |
| `BLOCKED` | `BLOCKED` | 风控/不可交易区 | 可以解释原因，不能生成交易计划 |
| `INVALIDATED` | `BLOCKED` | 失效/归档区 | 只能归档或等待重新进入链路 |
| `COOLDOWN` | `WAIT` | 冷却观察区 | 冷却期间不能展示为机会 |

## 禁止事项

- 禁止把 `CANDIDATE` 写成机会推荐。
- 禁止把 `EVIDENCE_SIGNAL` 写成买卖信号。
- 禁止把 `WAIT` 写成已触发。
- 禁止用榜单候选补狙击榜空位。
- 禁止用前端文案补齐后端没有生成的入场、止损、目标和失效条件。
